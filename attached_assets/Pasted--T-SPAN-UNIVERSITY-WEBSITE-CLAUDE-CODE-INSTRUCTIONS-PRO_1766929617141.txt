# T-SPAN UNIVERSITY WEBSITE - CLAUDE CODE INSTRUCTIONS

## PROJECT OVERVIEW
Build a modern, responsive website for T-SPAN University (Threat-Sovereign Platform for Autonomous Networks) - a global cybersecurity education institution. The website should attract professional registrations for their 10-weekend hybrid program launching March 15, 2025.

---

## TARGET AUDIENCE
- Cybersecurity professionals seeking advanced training
- Students interested in operational cybersecurity
- Academic faculty and researchers
- Enterprise partners
- Government stakeholders

---

## WEBSITE OBJECTIVES
1. Drive registrations before February 15, 2025 deadline
2. Communicate T-SPAN's unique value proposition (live infrastructure vs. theoretical training)
3. Establish credibility through institutional partnerships
4. Provide clear program information and pricing
5. Enable easy registration process

---

## TECHNICAL STACK RECOMMENDATION

### Option 1: Modern Single-Page Application (RECOMMENDED)
- **Framework**: React with Next.js 14+ (App Router)
- **Styling**: Tailwind CSS
- **Animations**: Framer Motion
- **Icons**: React Icons + Lucide React
- **Forms**: React Hook Form + Zod validation
- **Deployment**: Vercel (zero-config)

### Option 2: Lightweight Static Site
- **Base**: HTML5 + CSS3 + Vanilla JavaScript
- **Styling**: Custom CSS with CSS Variables
- **Animations**: AOS (Animate On Scroll)
- **Icons**: Font Awesome / Iconify
- **Forms**: HTML5 validation + EmailJS for submission

### Recommended Libraries

**Framer Motion (React):**


```bash
npm install aos
```

**GSAP (GreenSock) - Professional animations:**
```bash
npm install gsap
# Or CDN
<script src="https://cdn.jsdelivr.net/npm/gsap@3.12.5/dist/gsap.min.js"></script>
```

### Animation Guidelines

**Hero Section:**
- Fade in headline with slight upward movement
- Stagger statistics cards
- Pulse effect on CTA button
- Parallax background scroll

**Section Reveals:**
- Fade in on scroll (AOS or Framer Motion)
- Stagger children elements
- Slide in from left/right for comparison tables

**Interactive Elements:**
- Hover scale on cards (1.02-1.05)
- Smooth color transitions (200-300ms)
- Button ripple effects
- Icon rotations on hover

**Micro-interactions:**
- Form input focus animations
- Success/error state transitions
- Navigation menu smooth slide
- Tooltip appearances

**Performance:**
- Use CSS transforms (GPU-accelerated)
- Lazy load off-screen animations
- Reduce motion for accessibility (prefers-reduced-motion)

```javascript
// Countdown to February 15, 2025
const deadline = new Date('2025-02-15T23:59:59').getTime();

function updateCountdown() {
  const now = new Date().getTime();
  const distance = deadline - now;
  
  const days = Math.floor(distance / (1000 * 60 * 60 * 24));
  const hours = Math.floor((distance % (1000 * 60 * 60 * 24)) / (1000 * 60 * 60));
  const minutes = Math.floor((distance % (1000 * 60 * 60)) / (1000 * 60));
  const seconds = Math.floor((distance % (1000 * 60)) / 1000);
  
  return { days, hours, minutes, seconds };
}
```

## CONTENT WRITING GUIDELINES

### Voice & Tone
- **Professional yet accessible**: Avoid jargon overload
- **Confident and authoritative**: Backed by facts and partnerships
- **Action-oriented**: Clear CTAs throughout
- **Inclusive**: "Join us" not "we offer"
- **Urgent but not pushy**: Emphasize deadline without desperation

### Copywriting Best Practices
- **Headlines**: Benefit-driven, clear value proposition
- **Subheadlines**: Expand on headline, add credibility
- **Body**: Scannable, short paragraphs, bullet points
- **CTAs**: Action verbs (Register, Secure, Transform, Join)
- **Social Proof**: Numbers, partnerships, credentials

### Microcopy
- Form field placeholders: Helpful examples
- Button text: Specific (not just "Submit")
- Error messages: Friendly and actionable
- Success messages: Celebratory, next steps

### Registration Form Example
```jsx
// components/sections/Registration.jsx
import { useState } from 'react'
import { useForm } from 'react-hook-form'
import { zodResolver } from '@hookform/resolvers/zod'
import * as z from 'zod'

const formSchema = z.object({
  fullName: z.string().min(2, 'Name must be at least 2 characters'),
  email: z.string().email('Invalid email address'),
  phone: z.string().min(10, 'Invalid phone number'),
  country: z.string().min(1, 'Please select a country'),
  role: z.string().min(1, 'Please select your role'),
  motivation: z.string().min(200, 'Please write at least 200 characters'),
  consent: z.boolean().refine(val => val === true, 'You must agree to continue'),
})

export default function Registration() {
  const [isSubmitting, setIsSubmitting] = useState(false)
  const [submitSuccess, setSubmitSuccess] = useState(false)
  
  const { register, handleSubmit, formState: { errors } } = useForm({
    resolver: zodResolver(formSchema),
  })
  
  const onSubmit = async (data) => {
    setIsSubmitting(true)
    
    try {
      // Send to EmailJS or your API
      const response = await fetch('/api/register', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify(data),
      })
      
      if (response.ok) {
        setSubmitSuccess(true)
      }
    } catch (error) {
      console.error('Submission error:', error)
    } finally {
      setIsSubmitting(false)
    }
  }
  
  if (submitSuccess) {
    return (
      <div className="text-center py-20">
        <h2 className="text-4xl font-bold text-white mb-4">
          Application Submitted! ðŸŽ‰
        </h2>
        <p className="text-gray-300">
          We'll review your application and contact you within 48 hours.
        </p>
      </div>
    )
  }
  
  return (
    <section id="register" className="py-20 bg-dark">
      <div className="container mx-auto px-4 max-w-3xl">
        <h2 className="text-4xl md:text-5xl font-bold text-center text-white mb-12">
          Secure Your Spot â€“ Register by February 15, 2025
        </h2>
        
        <form onSubmit={handleSubmit(onSubmit)} className="space-y-6">
          {/* Form fields */}
          <div>
            <label className="block text-white mb-2">Full Name *</label>
            <input
              {...register('fullName')}
              className="w-full px-4 py-3 rounded-lg bg-white/5 border border-white/10 text-white placeholder-gray-500 focus:border-primary focus:outline-none"
              placeholder="John Doe"
            />
            {errors.fullName && (
              <p className="mt-1 text-sm text-red-400">{errors.fullName.message}</p>
            )}
          </div>
          
          {/* Add all other form fields similarly */}
          
          <button
            type="submit"
            disabled={isSubmitting}
            className="w-full btn-primary py-4 text-lg disabled:opacity-50 disabled:cursor-not-allowed"
          >
            {isSubmitting ? 'Submitting...' : 'Submit Application'}
          </button>
        </form>
      </div>
    </section>
  )
}
```
